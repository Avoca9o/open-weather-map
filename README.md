# open-weather-map

## Выводы по распараллеливанию анализа данных
Код в analyze_data.ipynb

С использованием библиотеки joblib с 8ю воркерами обработка датасета занимает примерно в 8 раз меньше времени (меньше потому что появляются накладные расходы на создание этих самых ворекров и объединение конечного датасета)

## Выводы по синхронному/асинхронному взаимодействию с open weather map API
Код в open_weather_map.ipynb

Асинхронное взаимодействие заметно быстрее. Анализировал на примере 10 вызовов api. Также библиотека aiohttp заметно быстрее библиотеки requests, потому что даже в среднем на запрос тратится сильно больше времени, чем на 10 параллельных в случае с aiohttp.

## Приложение
Код в app.py.
Само приложение развернуто на домене https://open-weather-map-vladimirov.streamlit.app/
Подразумевается, что на вход приложению подается датасет, сгенерированный по примеру из файла с ДЗ. Какие-то проверки стоят, но может быть покрыто не все. Также пример файла, который можно использовать для тестирования приложения -- temperature_data.csv из текущего репозитория.
